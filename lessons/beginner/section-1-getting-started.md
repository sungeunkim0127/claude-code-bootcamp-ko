## 레슨 1: Claude Code란 무엇인가?

### 학습 목표

### Claude Code 소개

Claude Code는 Anthropic에서 개발한 **에이전틱 코딩 도우미**입니다. 단순한 코드 자동완성이 아니라, 실제로 파일을 읽고, 쓰고, 편집하고, 명령을 실행할 수 있는 **자율적인 AI 어시스턴트**입니다.

### 주요 특징

| 특징 | 설명 |
|------|------|
| **에이전틱** | 스스로 도구를 사용하여 작업 수행 |
| **파일 작업** | 읽기, 쓰기, 편집, 검색 |
| **명령 실행** | 터미널 명령 실행 가능 |
| **컨텍스트 이해** | 프로젝트 전체 맥락 파악 |

### Claude 모델 종류

| 모델 | 특징 | 용도 |
|------|------|------|
| **Haiku** | 빠르고 경제적 | 간단한 작업 |
| **Sonnet** | 균형잡힌 성능 | 일반 개발 작업 |
| **Opus** | 최고 성능 | 복잡한 작업 |

### ChatGPT와의 차이점

```
ChatGPT:
  사용자: "파일 읽어줘"
  ChatGPT: "저는 파일을 직접 읽을 수 없습니다..."

Claude Code:
  사용자: "파일 읽어줘"
  Claude: [실제로 Read 도구를 사용하여 파일 읽음]
  Claude: "파일 내용은 다음과 같습니다..."
```

### 설치 요구사항

```bash

# npm 또는 yarn 필요

```

### 설치 방법

```bash

npm install -g @anthropic-ai/claude-code

# 설치 확인
claude --version
```

### 계정 유형

| 플랜 | 특징 |
|------|------|
| **Pro** | 개인 사용자용 |
| **Max** | 고급 기능 포함 |
| **Teams** | 팀 협업 기능 |
| **Enterprise** | 기업용 관리 기능 |

### 첫 실행

```bash
# Claude Code 시작
claude

# 도움말 보기
```

| 옵션 | 설명 |
|------|------|
| `--help` | 도움말 표시 |
| `--version` | 버전 확인 |

| `--resume` | 이전 세션 재개 |

## 레슨 3: 첫 번째 대화

### 학습 목표
- Claude와 대화 시작하기
- 기본 명령어 사용하기 (/help, /clear, /exit)
- 명확한 지시 방법 배우기
- 프로젝트 분석 요청하기

### 세션 시작하기

```bash
# 프로젝트 폴더로 이동
cd 프로젝트폴더

# Claude 시작
claude
```

### 기본 인사

```
사용자: 안녕, 넌 뭘 할 수 있어?

Claude: 안녕하세요! 저는 Claude Code입니다. 다음과 같은 작업을 도와드릴 수 있습니다:

1. **파일 작업**: 읽기, 쓰기, 편집
2. **코드 검색**: 패턴 검색, 파일 찾기
3. **명령 실행**: npm, git 등 터미널 명령
4. **코드 분석**: 설명, 리뷰, 개선 제안
5. **Git 작업**: 커밋, 브랜치, PR 생성
```

### 세션 내 명령어

| 명령어 | 기능 |
|--------|------|
| `/help` | 도움말 보기 |
| `/clear` | 대화 초기화 |
| `/exit` | 세션 종료 |
| `/compact` | 컨텍스트 압축 |

### 효과적인 질문 방법

**❌ 모호한 질문:**
```
"코드 봐줘"
"이거 고쳐줘"
```

**✅ 명확한 질문:**
```
"src/utils/api.js 파일을 읽고 각 함수가 하는 일을 설명해줘"
"handleSubmit 함수에 에러 처리를 추가해줘"
```

### 프로젝트 분석 요청

```
사용자: 현재 디렉토리를 분석하고 프로젝트 구조를 설명해줘

Claude: [Glob와 Read 도구를 사용하여 분석]
이 프로젝트는 다음과 같은 구조입니다:
- src/: 소스 코드
- tests/: 테스트 파일
- package.json: 의존성 관리
...
```

### 연습 3.1: 첫 대화

1. 터미널에서 아무 프로젝트 폴더로 이동
2. `claude` 실행

   ```
   "package.json 읽어줘" (있다면)
   "/help 보여줘"
   ```
4. `/exit`으로 종료

**확인사항**:

## 레슨 4: 권한 이해하기

### 학습 목표
- 권한 모드 종류 이해하기
- Claude가 권한을 요청하는 이유 알기
- 권한 승인/거부 방법 배우기
- 권한 설정하기

### 왜 권한이 필요한가?

Claude Code는 실제로 파일을 수정하고 명령을 실행합니다. 안전을 위해 민감한 작업 전에 사용자 확인을 요청합니다.

### 권한 모드

| 모드 | 설명 | 사용 시점 |
|------|------|----------|
| **normal** | 각 작업마다 확인 | 학습 중, 신중한 작업 |
| **auto-accept** | 허용된 작업 자동 승인 | 신뢰할 수 있는 작업 |
| **plan** | 읽기 전용 탐색 | 계획 수립 단계 |

### 권한 요청 예시

```
Claude: 다음 작업을 수행하려고 합니다:
        [Edit] src/app.js에서 문자열 변경

        승인하시겠습니까? [Y/n/always/never]
```

### 응답 옵션

| 입력 | 의미 |
|------|------|
| `Y` 또는 `Enter` | 이번만 승인 |
| `n` | 이번만 거부 |
| `always` | 이 유형 항상 승인 |
| `never` | 이 유형 항상 거부 |

### 권한 설정

```json
// ~/.claude/settings.json
{
  "permissions": {
    "allow": [
      "Read",
      "Glob",
      "Grep",
      "Bash(npm test*)",
      "Bash(git status)"
    ],
    "deny": [
      "Bash(rm -rf *)"
    ]
  }
}
```

### 연습 4.1: 권한 체험

1. Claude 세션 시작
2. "새 파일 test.txt를 만들어줘"라고 요청
3. 권한 요청을 주의 깊게 읽기
4. 'Y'로 승인
5. 파일이 생성되었는지 확인

- [ ] 권한 요청이 표시됨
- [ ] 승인 후 작업이 완료됨

## 레슨 5: 기본 워크플로우 패턴

### 학습 목표
- 탐색 → 계획 → 코드 → 검증 패턴 배우기
- Claude의 작업 분해 방식 이해하기
- 변경사항 검토 방법 배우기
- /resume으로 세션 재개하기

### 기본 워크플로우

```
┌─────────────────────────────────────┐
│          1. 탐색 (Explore)          │
│    코드베이스 이해, 파일 읽기       │
└─────────────────────────────────────┘
                  ↓
┌─────────────────────────────────────┐
│          2. 계획 (Plan)             │
│    접근 방식 결정, 영향 분석        │
└─────────────────────────────────────┘
                  ↓
┌─────────────────────────────────────┐
│          3. 코드 (Code)             │
│    변경사항 구현, 편집              │
└─────────────────────────────────────┘
                  ↓
┌─────────────────────────────────────┐
│          4. 검증 (Verify)           │
│    테스트 실행, 변경사항 확인       │
└─────────────────────────────────────┘
```

### 워크플로우 예시

```
사용자: 로그인 기능에 비밀번호 유효성 검사를 추가해줘

Claude:
1. [탐색] 먼저 현재 로그인 코드를 확인하겠습니다.
   [Read: src/auth/login.js]

2. [계획] 다음과 같이 수정하겠습니다:
   - validatePassword 함수 추가
   - 로그인 전 유효성 검사 호출

3. [코드] 변경사항을 적용합니다.
   [Edit: src/auth/login.js]

4. [검증] 테스트를 실행하겠습니다.
   [Bash: npm test]
```

### 변경사항 검토

Claude가 Edit을 요청하면:
- 변경 전/후 내용이 표시됨
- 차이점을 확인한 후 승인

### 세션 재개

```bash
# 세션 이름 지정하여 시작
claude --session "로그인-기능"

# 작업 후 종료
/exit

# 나중에 재개
claude --resume "로그인-기능"
```

### 연습 5.1: 워크플로우 실습

1. 새 세션 시작: `claude --session "연습"`
2. 간단한 기능 추가 요청:
   ```

   ```
3. Claude의 워크플로우 관찰
4. `/exit`으로 종료
5. `claude --resume "연습"`으로 재개
6. 이전 컨텍스트가 유지되는지 확인

**확인사항**:
- [ ] 워크플로우 단계를 관찰함
- [ ] 세션을 성공적으로 재개함
