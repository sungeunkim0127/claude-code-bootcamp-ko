## 레슨 6: Read 도구로 파일 읽기

### 학습 목표
- Read 도구의 사용법 이해하기
- 파일 참조 방법 배우기
- @-멘션 활용하기
- 여러 파일 효율적으로 읽기

### Read 도구란?

Read 도구는 파일 내용을 읽어옵니다. Claude가 코드를 이해하고 분석하는 데 필수적입니다.

### 기본 사용법

```
"app.js 파일 읽어줘"
"src/utils/helper.ts 내용 보여줘"
"package.json 분석해줘"
```

### @-멘션으로 파일 참조

```
"@src/App.tsx 설명해줘"
"@config/database.js:10-30 이 부분 설명해줘"
```

### 라인 범위 지정

```
"@utils/api.js:1-50 읽어줘"      # 1-50 라인
"@app.js:100 주변 코드 보여줘"    # 100번 라인 주변
```

### 여러 파일 읽기

```
"@src/App.tsx와 @src/index.tsx 둘 다 읽어줘"
"src 폴더의 모든 컴포넌트 파일 분석해줘"
```

### 연습 6.1: 파일 읽기 실습

1. 프로젝트 폴더에서 Claude 시작
2. 다음 요청 시도:
   ```
   "package.json 읽고 의존성 목록 보여줘"
   "@src/index.js 설명해줘"
   ```

**확인사항**:
- [ ] 파일 내용이 표시됨
- [ ] @-멘션이 작동함
- [ ] Claude가 내용을 설명함

## 레슨 7: Write 도구로 파일 생성

### 학습 목표
- Write vs Edit 차이 이해하기
- 새 파일 생성하기
- 파일 경로 처리 이해하기
- 작성 전 내용 검토하기

### Write 도구란?

Write 도구는 **새 파일을 생성**하거나 **기존 파일을 완전히 덮어씁니다**.

### 언제 Write를 사용하나요?

| 상황 | 도구 |
|------|------|
| 새 파일 생성 | Write ✅ |
| 기존 파일 일부 수정 | Edit ✅ |
| 기존 파일 전체 교체 | Write ✅ |

### 사용 예시

```
"hello.js 파일을 만들고 Hello World를 출력하는 코드 작성해줘"

"utils/logger.ts 파일을 새로 만들어줘:
- info, warn, error 함수 포함
- 타임스탬프 추가"
```

### 경로 처리

```
"components/Button.tsx 생성해줘"  # 폴더가 없으면 생성됨
"src/utils/helper.js 만들어줘"
```

### 작성 전 확인

Claude가 Write를 실행하기 전에:
1. 생성될 파일 경로 확인
2. 내용 미리보기 확인
3. 승인 후 생성

### 연습 7.1: 파일 생성 실습

1. Claude에게 요청:
   ```
   "utils/greet.js 파일을 만들어줘.
    이름을 받아서 '안녕하세요, {이름}님!'을 반환하는
    greet 함수를 export 해줘"
   ```
2. 권한 요청 확인
3. 파일 생성 확인

**확인사항**:
- [ ] 파일이 생성됨
- [ ] 내용이 올바름
- [ ] export가 정상 작동함

## 레슨 8: Edit 도구로 파일 편집

### 학습 목표
- 정확한 문자열 치환 이해하기
- Edit 도구 모범 사례 배우기
- 여러 줄 편집하기
- replace_all로 이름 변경하기

### Edit 도구란?

Edit 도구는 **기존 파일의 특정 부분을 정확히 교체**합니다.

### 작동 방식

```
1. 기존 문자열 찾기 (정확히 일치해야 함)
2. 새 문자열로 교체
3. 파일 저장
```

### 사용 예시

```
"handleClick 함수 이름을 onClick으로 변경해줘"
"console.log 대신 logger.info를 사용하도록 수정해줘"
"validateEmail 함수에 null 체크 추가해줘"
```

### 주의사항

**흔한 실패 원인:**
```
❌ 공백이나 탭이 다름
❌ 찾을 문자열이 파일에 없음
❌ 여러 곳에 같은 문자열이 있음
```

**해결 방법:**
```
✅ Read로 먼저 정확한 내용 확인
✅ 더 많은 컨텍스트 포함
✅ replace_all 사용 (전체 교체 시)
```

### replace_all 사용

```
"oldFunction을 newFunction으로 파일 전체에서 변경해줘"
"모든 console.log를 logger.debug로 교체해줘"
```

### 연습 8.1: 편집 실습

1. 앞서 만든 greet.js 읽기
2. 편집 요청:
   ```
   "greet 함수를 수정해서 시간에 따라
    '좋은 아침', '안녕하세요', '좋은 저녁'으로
    인사하도록 바꿔줘"
   ```
3. 변경사항 확인

**확인사항**:
- [ ] Edit 도구가 사용됨
- [ ] 변경 전/후 diff 확인
- [ ] 수정이 올바르게 적용됨

## 레슨 9: Glob으로 파일 찾기

### 학습 목표
- Glob 패턴 사용하기
- 일반적인 패턴 배우기
- 다른 도구와 조합하기
- Glob vs Grep 구분하기

### Glob이란?

Glob은 **패턴으로 파일을 찾습니다**. 파일 이름이나 경로로 검색할 때 사용합니다.

### 기본 패턴

| 패턴 | 의미 |
|------|------|
| `*` | 모든 문자 (폴더 제외) |
| `**` | 모든 폴더 |
| `?` | 단일 문자 |
| `{a,b}` | a 또는 b |

### 실용적인 예시

```
"모든 TypeScript 파일 찾아줘"
→ **/*.ts

"src 폴더의 React 컴포넌트 찾아줘"
→ src/**/*.tsx

"설정 파일들 목록 보여줘"
→ *.config.{js,ts,json}

"테스트 파일 찾아줘"
→ **/*.test.ts, **/*.spec.ts
```

### 자주 쓰는 패턴

```bash
**/*.js           # 모든 JS 파일
src/**/*.tsx      # src 하위 모든 TSX
*.{js,ts}         # 루트의 JS 또는 TS
!node_modules/**  # node_modules 제외
```

```

"JSON 설정 파일 목록 보여줘"
"components 폴더의 모든 파일 찾아줘"
```

**확인사항**:

## 레슨 10: Grep으로 내용 검색

### 학습 목표
- 정규식으로 내용 검색하기
- 출력 모드 사용하기 (content, files_with_matches, count)
- 컨텍스트 플래그 활용하기 (-A, -B, -C)
- 파일 유형으로 필터링하기

### Grep이란?

Grep은 **파일 내용에서 패턴을 검색**합니다.

### 기본 사용법

```
"TODO 주석 찾아줘"
"useState 사용하는 곳 찾아줘"
"fetchAPI 함수 정의 찾아줘"
```

### 출력 모드

| 모드 | 결과 |
|------|------|
| `files_with_matches` | 파일 이름만 (기본값) |
| `content` | 일치하는 라인 내용 |
| `count` | 매치 개수 |

### 컨텍스트 옵션

```
"handleError 찾아서 앞뒤 5줄씩 보여줘"
→ -C 5 (컨텍스트 5줄)

"import문 찾아서 그 아래 3줄 보여줘"
→ -A 3 (뒤 3줄)
```

### 정규식 예시

```
function\s+\w+       # 함수 선언
import.*from        # import 문
console\.(log|warn) # console.log 또는 warn
TODO|FIXME|HACK     # 주석 태그
```

```

"async 함수 정의 찾아서 내용 보여줘"
"import React 하는 곳 개수 세줘"
```

**확인사항**:

- [ ] 컨텍스트 옵션을 이해함

## 레슨 11: Bash로 명령 실행

### 학습 목표
- 터미널 명령 실행하기
- Bash vs 전용 도구 구분하기
- 명령 체이닝하기
- 실패 처리하기

### Bash 도구란?

Bash 도구는 **터미널 명령을 실행**합니다.

### 일반적인 사용 사례

```
"npm test 실행해줘"
"git status 보여줘"
"현재 Node 버전 확인해줘"
"의존성 설치해줘"
```

### 언제 Bash를 사용하나요?

| 작업 | 도구 |
|------|------|
| 파일 읽기 | Read ✅ (Bash cat ❌) |
| 파일 검색 | Glob ✅ (Bash find ❌) |
| 내용 검색 | Grep ✅ (Bash grep ❌) |
| npm/git 명령 | Bash ✅ |
| 빌드/테스트 | Bash ✅ |

### 명령 체이닝

```
"테스트 실행하고 빌드해줘"
→ npm test && npm run build

"린트 실행하고 수정해줘"
→ npm run lint -- --fix
```

### 연습 11.1: 명령 실행 실습

```

"npm run test 실행해줘" (package.json에 있다면)
"현재 디렉토리 경로 알려줘"
```

## 레슨 12: 도구 조합하기

### 학습 목표
- 도구 체이닝 패턴 배우기
- 병렬 vs 순차 실행 이해하기
- 여러 도구를 한 요청에 사용하기
- 효율적인 도구 사용 최적화하기

### 도구 체이닝 패턴

```
Grep → Read → Edit

1. Grep: "handleError 함수 찾아줘"
2. Read: 해당 파일 읽기
3. Edit: 수정하기
```

### 병렬 실행

독립적인 작업은 동시에 실행됩니다:

```
"package.json 읽고, git status 확인하고,
 모든 테스트 파일 찾아줘"

→ 세 작업이 병렬로 실행됨
```

### 순차 실행

의존적인 작업은 순서대로:

```
"테스트 파일 찾아서 각각 분석해줘"

→ 먼저 찾고, 그 다음 분석
```

### 효율적인 요청

**비효율적:**
```
"app.js 읽어줘"
"utils.js 읽어줘"
"config.js 읽어줘"
(3개의 별도 요청)
```

**효율적:**
```
"@app.js, @utils.js, @config.js 읽어줘"
(1개의 요청, 병렬 실행)
```

```
Claude의 작업 과정:
1. Grep으로 console.log 찾기
2. 각 파일 Read하기
3. Edit으로 수정하기
```
