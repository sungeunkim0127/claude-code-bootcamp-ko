# 자주 묻는 질문 (FAQ)

## 시작하기

**Q: 프로그래밍 경험이 필요한가요?**
A: 기본적인 프로그래밍 지식이 있으면 도움이 되지만, 부트캠프는 처음부터 시작합니다. 터미널을 사용하고 코드를 편집할 수 있다면 준비된 것입니다.

**Q: 완료하는 데 얼마나 걸리나요?**
A: 선택한 경로에 따라 다릅니다:
- 빠른 시작: 2주 파트타임 (30-40시간)
- 전문가 과정: 4주 집중 (80-100시간)
- 마스터 과정: 6-8주 집중 (160-200시간)

**Q: 파트타임으로 학습할 수 있나요?**
A: 물론입니다! 자신의 속도에 맞춰 일정을 조정하세요. 속도보다 꾸준함이 중요합니다.

**Q: 어떤 경로를 선택해야 하나요?**
A:
- Claude Code가 처음이라면? → 빠른 시작 또는 전문가 과정
- 숙련된 개발자라면? → 전문가 또는 마스터 과정
- 팀 리더라면? → 전문가 과정 후 기업 섹션 집중
- DevOps 담당이라면? → DevOps 특화 경로 따라가기

## 기술적인 질문

**Q: Read, Write, Edit의 차이점은 무엇인가요?**
A:
- **Read**: 파일 내용 보기
- **Write**: 새 파일 생성 또는 기존 파일 완전 덮어쓰기
- **Edit**: 기존 파일의 특정 부분만 정확히 수정
항상 수정할 때는 Edit, 새 파일은 Write를 사용하세요.

**Q: 플랜 모드는 언제 사용하나요?**
A: 플랜 모드는 다음 상황에 적합합니다:
- 복잡한 리팩토링
- 대규모 기능 개발
- 아키텍처 변경
- 실행 전 접근 방식을 검토하고 싶을 때

**Q: 스킬, 훅, 서브에이전트의 차이점은 무엇인가요?**
A:
- **스킬**: 수동으로 호출하는 워크플로우 (`/스킬명`)
- **훅**: 이벤트에 자동으로 반응하는 동작 (저장 후 자동 포맷 등)
- **서브에이전트**: 독립적인 컨텍스트를 가진 특화된 AI 어시스턴트

**Q: Claude가 어떤 도구를 사용하는지 어떻게 알 수 있나요?**
A: Claude는 대화 중에 도구 호출을 표시합니다. `[Tool: Read - file.js]` 등으로 보입니다.

## 학습 및 진행

**Q: 레슨을 건너뛸 수 있나요?**
A: 가능하지만, 각 레슨은 이전 내용을 기반으로 합니다. 건너뛰면 이해에 공백이 생길 수 있습니다.

**Q: 다음 단계로 넘어갈 준비가 됐는지 어떻게 알 수 있나요?**
A: 각 섹션 끝의 확인 체크리스트를 완료하세요. 모든 항목을 자신 있게 할 수 있다면 다음으로 넘어가세요.

**Q: 연습문제에서 막히면 어떻게 하나요?**
A:
1. 레슨 목표를 다시 읽으세요
2. TROUBLESHOOTING.md 확인하세요
3. 더 간단한 버전으로 먼저 시도하세요
4. Claude에게 도움을 요청하세요
5. 일단 넘어가고 나중에 다시 돌아오세요

**Q: 20개 프로젝트를 모두 해야 하나요?**
A: 최소한 각 레벨당 하나씩은 하세요. 더 많이 연습할수록 더 잘 익힙니다.

## 문제 해결

**Q: Edit 도구가 "string not found" 에러를 냅니다**
A: 가장 흔한 원인은 공백 불일치입니다. Read 출력에서 정확한 문자열을 복사하세요. 모든 공백과 탭을 포함해서요.

**Q: Claude가 컨텍스트를 잊어버리는 것 같습니다**
A:
- `/compact`로 대화 압축하기
- 이전 결정을 명시적으로 언급하기
- CLAUDE.md에 지속적인 컨텍스트 저장
- 새 주제는 새 세션 시작

**Q: Claude의 변경사항을 되돌리려면 어떻게 하나요?**
A: Git을 사용 중이라면: `git checkout 파일명`
Git 없이는 쉽게 되돌릴 수 없습니다 - 항상 Git을 사용하세요!

**Q: Claude가 느립니다**
A:
- 간단한 작업에는 Haiku 모델 사용
- `/compact`로 컨텍스트 줄이기
- 큰 요청을 작은 것으로 나누기
- 병렬 작업 요청하기

## 고급 주제

**Q: 커스텀 MCP 서버는 어떻게 만드나요?**
A: 레슨 59와 프로젝트 15를 참조하세요. TypeScript/JavaScript 지식과 MCP 프로토콜 이해가 필요합니다.

**Q: CI/CD에서 Claude Code를 사용할 수 있나요?**
A: 네! 레슨 89와 프로젝트 18에서 GitHub Actions 통합을 다룹니다.

**Q: 팀과 스킬을 공유하려면 어떻게 하나요?**
A:
1. skills 디렉토리 패키징
2. Git 저장소로 공유
3. 팀원이 `~/.claude/skills/` 또는 프로젝트 `.claude/skills/`에 복사
4. 또는 기업 플러그인 생성

**Q: 사용자 스킬과 프로젝트 스킬의 차이점은?**
A:
- **사용자** (`~/.claude/skills/`): 모든 프로젝트에서 사용 가능
- **프로젝트** (`.claude/skills/`): 이 프로젝트에서만, Git으로 공유
